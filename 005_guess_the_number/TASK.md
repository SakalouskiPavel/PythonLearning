# Задача 005 — «Угадай число»

## Цель

Создать первую консольную игру и изучить цикл `while`, управление циклом через
`break` и `continue`, а также использование модуля стандартной библиотеки.

## Условие

Создай файл `main.py`. При запуске программа должна загадать случайное целое
число от `1` до `100` включительно и предложить пользователю его угадать.

После каждой попытки программа сообщает:

- `Загаданное число больше.`, если введённое число слишком маленькое;
- `Загаданное число меньше.`, если введённое число слишком большое;
- сообщение о победе и количество попыток, если число угадано.

После правильного ответа игра завершается.

Пример одного из возможных диалогов:

```text
Я загадал число от 1 до 100.
Твоя попытка: 50
Загаданное число больше.
Твоя попытка: 75
Загаданное число меньше.
Твоя попытка: 63
Верно! Число угадано за 3 попытки.
```

Поскольку число случайное, конкретный диалог при каждом запуске будет отличаться.

## Проверка диапазона

Если пользователь ввёл число меньше `1` или больше `100`, выведи предупреждение
и запроси число снова. Такая попытка не должна увеличивать счётчик попыток.

Пока можно считать, что пользователь всегда вводит целое число. Обработку текста
вместо числа мы изучим позже.

## Темы и ключевые слова для изучения

- цикл `while`;
- бесконечный цикл `while True`;
- операторы `break` и `continue`;
- счётчик попыток и оператор `+=`;
- импорт модулей: `import random`;
- `random.randint(a, b)`;
- стандартная библиотека Python;
- повторное использование `if` / `elif` / `else`;
- логический оператор `or`;
- границы диапазона;
- жизненный цикл одной итерации цикла.

## Ограничения

- Случайное число должно создаваться ровно один раз до начала цикла.
- Используй `random.randint(1, 100)`.
- Для запросов попыток используй один цикл `while`.
- Не используй цикл `for`.
- При победе выйди из цикла с помощью `break`.
- Для значения вне диапазона используй `continue`.
- Увеличивай счётчик только для значений от `1` до `100` включительно.
- Не показывай загаданное число пользователю до победы.
- Используй имена переменных в стиле `snake_case`.
- Не используй сторонние библиотеки.

## Критерии готовности

- Файл называется `main.py` и запускается без ошибок.
- При каждом запуске загадывается число в диапазоне от `1` до `100`.
- Игра продолжает запрашивать числа до правильного ответа.
- Подсказки «больше» и «меньше» соответствуют загаданному числу.
- После победы цикл и программа завершаются.
- Количество засчитанных попыток вычисляется правильно.
- Значения `0`, `-10` и `101` отклоняются и не засчитываются.
- В решении есть ровно один цикл `while`, а случайное число создаётся вне него.
- Ты можешь объяснить разницу между `break` и `continue`.

## Как проверить случайную игру

Сыграй несколько раз. В одной из игр намеренно введи сначала `0` и `101`, чтобы
проверить диапазон. Для быстрого поиска числа используй стратегию деления
диапазона пополам: начни с `50`, а затем каждый раз выбирай середину оставшейся
части диапазона.

## Дополнительное задание (необязательно)

Для каждой неправильной, но допустимой попытки вычисли расстояние до загаданного
числа с помощью `abs(guess - secret_number)`:

- если расстояние не больше `5`, дополнительно выведи `Очень близко!`;
- если расстояние от `6` до `15`, выведи `Тепло.`;
- иначе выведи `Холодно.`.

Основные подсказки «больше» и «меньше» должны сохраниться.
