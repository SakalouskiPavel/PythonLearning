# Задача 009 — «Телефонная книга с сохранением»

## Цель

Научиться читать и записывать файлы, сохранять структуры Python в формате JSON и
обрабатывать ожидаемые ошибки. После этой задачи данные программы не будут
исчезать при каждом перезапуске.

## Основа задачи

Создай файл `main.py`. Возьми за основу свою телефонную книгу из задачи 008:
команды `add`, `find`, `show`, `remove`, `update` и `exit`, а также функции работы
со словарём должны сохраниться.

Главное изменение: контакты нужно загружать из файла при запуске и сохранять
после каждого успешного изменения.

## Расположение файла данных

Контакты хранятся в файле `contacts.json` рядом с `main.py`. Чтобы путь не зависел
от папки, из которой запущена программа, используй `pathlib`:

```python
from pathlib import Path

DATA_FILE = Path(__file__).with_name("contacts.json")
```

`DATA_FILE` является константой, поэтому записывается в верхнем регистре.

## Формат JSON

Файл должен содержать JSON-объект, соответствующий словарю контактов:

```json
{
  "Анна": "+375291112233",
  "Макс": "+375292223344"
}
```

Не формируй этот текст вручную — используй стандартный модуль `json`.

## Обязательные функции для файлов

### `load_contacts(file_path)`

Функция получает путь и:

1. Открывает файл на чтение с `encoding="utf-8"` через `with open(...)`.
2. Возвращает данные, полученные через `json.load()`.
3. Если файла ещё нет (`FileNotFoundError`), возвращает пустой словарь.
4. Если JSON повреждён (`json.JSONDecodeError`), выводит предупреждение и
   возвращает пустой словарь вместо аварийного завершения.
5. Если JSON корректен, но верхний уровень не является словарём, также выводит
   предупреждение и возвращает пустой словарь. Для проверки используй
   `isinstance(data, dict)`.

### `save_contacts(contacts, file_path)`

Функция открывает файл на запись с `encoding="utf-8"` и сохраняет словарь через:

```python
json.dump(contacts, file, ensure_ascii=False, indent=2)
```

Функция ничего не возвращает и не взаимодействует с `input()`.

## Изменения в `main()`

- В начале `main()` вместо пустого словаря вызови `load_contacts(DATA_FILE)`.
- После успешного `add` вызови `save_contacts(contacts, DATA_FILE)`.
- Сделай то же после успешных `remove` и `update`.
- Не сохраняй файл, если операция не изменила словарь.
- Команда `exit` завершает программу без дополнительного сохранения: каждое
  изменение уже должно быть записано на диск.

Остальное поведение телефонной книги соответствует задаче 008. Контакты при
`show` по-прежнему выводятся в алфавитном порядке.

## Темы и ключевые слова для изучения

- файлы и режимы открытия `"r"` и `"w"`;
- менеджер контекста `with`;
- кодировка `encoding="utf-8"`;
- модуль `json`;
- `json.load()` и `json.dump()`;
- сериализация и десериализация;
- исключения `FileNotFoundError` и `json.JSONDecodeError`;
- конструкция `try` / `except`;
- проверка типа через `isinstance()`;
- модуль `pathlib`, класс `Path` и `__file__`;
- константы и стиль `UPPER_SNAKE_CASE`;
- постоянное хранение данных (persistence).

## Ограничения

- Используй стандартные модули `json` и `pathlib`; сторонние библиотеки не нужны.
- Путь к данным создай относительно `__file__`, а не текущей рабочей папки.
- Читай и записывай файл только через `with open(...)`.
- Всегда указывай UTF-8.
- Используй `json.load()` и `json.dump()`, а не ручное создание JSON-строки.
- Обрабатывай отсутствующий и повреждённый файл только в `load_contacts()`.
- Не используй голый `except:` и не перехватывай общий `Exception`.
- Логика добавления, поиска, удаления и обновления остаётся в отдельных функциях.
- Сохраняй данные сразу после каждой успешной изменяющей операции.
- Используй `main()` и защиту `if __name__ == "__main__":`.

## Критерии готовности

- Первый запуск без `contacts.json` не вызывает ошибку и показывает пустую книгу.
- После добавления создаётся читаемый UTF-8 JSON-файл.
- Кириллические имена сохраняются обычными буквами, а не кодами `\uXXXX`.
- После завершения и нового запуска контакты восстанавливаются.
- Удаление и обновление отражаются в файле сразу.
- Дубликат и неуспешное удаление не вызывают ненужного сохранения.
- Повреждённый JSON не завершает программу с traceback.
- Корректный JSON-массив, например `[]`, отклоняется как неверный тип данных.
- Функции загрузки и сохранения можно импортировать без запуска меню.
- Ты можешь объяснить, зачем нужны `with`, `ensure_ascii=False` и обработка
  `FileNotFoundError`.

## Сценарий самопроверки

1. Убедись, что `contacts.json` отсутствует, и запусти `show`, затем `exit`.
2. Запусти программу снова, добавь `Анна: 111` и `Макс: 222`, затем заверши её.
3. Открой `contacts.json` и проверь формат и кириллицу.
4. Запусти программу третий раз и выполни `show`: оба контакта должны загрузиться.
5. Обнови телефон Анны на `999`, удали Макса и перезапусти программу.
6. Убедись, что осталась только `Анна: 999`.
7. Временно замени содержимое файла на `{broken`, запусти программу и проверь
   предупреждение без traceback.
8. Проверь также содержимое `[]`: оно является корректным JSON, но неверным типом
   для телефонной книги.

Файлы `contacts.json` и `contacts.txt` уже добавлены в корневой `.gitignore`,
поэтому тестовые личные данные не попадут в Git.
